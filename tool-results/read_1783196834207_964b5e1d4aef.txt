     1тЖТ     1тЖТ# LeetCode Tracker тАФ Full-Stack Build Prompt
     2тЖТ     2тЖТ
     3тЖТ     3тЖТ> **Stack:** Python ┬╖ FastAPI ┬╖ React (Vite + TypeScript) ┬╖ MongoDB Atlas (cloud) ┬╖ JWT auth
     4тЖТ     4тЖТ> **Goal:** Rebuild the existing client-side LeetCode Tracker as a multi-user full-stack app. Users register/login, own their own sheets, upload CSVs, and track solved progress per sheet тАФ all persisted server-side.
     5тЖТ     5тЖТ> **Tone:** Production-ready, deployable to Render/Railway (backend) + Vercel (frontend). Not a toy.
     6тЖТ     6тЖТ
     7тЖТ     7тЖТ---
     8тЖТ     8тЖТ
     9тЖТ     9тЖТ## 0. Project Context
    10тЖТ    10тЖТ
    11тЖТ    11тЖТThe current app is a single-user, 100% client-side Next.js prototype. Data lives in `localStorage`. You are rebuilding it as a **proper multi-user SaaS-style app** with:
    12тЖТ    12тЖТ
    13тЖТ    13тЖТ- A **FastAPI** backend exposing a REST API
    14тЖТ    14тЖТ- A **MongoDB Atlas** cloud database (free M0 tier is fine)
    15тЖТ    15тЖТ- **JWT**-based authentication (access + refresh tokens)
    16тЖТ    16тЖТ- A **React + Vite + TypeScript** frontend (single-page app, dark-themed, GitHub ├Ч Linear ├Ч Vercel aesthetic)
    17тЖТ    17тЖТ- Per-user ownership: every sheet belongs to exactly one user; users cannot see or mutate each other's data
    18тЖТ    18тЖТ
    19тЖТ    19тЖТ### Functional parity with the existing prototype
    20тЖТ    20тЖТ
    21тЖТ    21тЖТThe new app MUST preserve every feature of the current prototype:
    22тЖТ    22тЖТ
    23тЖТ    23тЖТ1. CSV upload with header validation (`ID, URL, Title, Difficulty, Acceptance %, Frequency %`)
    24тЖТ    24тЖТ2. Multi-sheet support тАФ each upload becomes a named sheet (e.g. `DP`, `Arrays`, `Graphs`)
    25тЖТ    25тЖТ3. Per-sheet question list + per-sheet solved state
    26тЖТ    26тЖТ4. Filter sidebar: title search, multi-select difficulty, dual-handle frequency range + preset chips (100/75/50/25%), dual-handle acceptance range, solved toggle
    27тЖТ    27тЖТ5. Sortable, paginated table (25/50/100 rows per page)
    28тЖТ    28тЖТ6. Active-filters chip row + "Clear all"
    29тЖТ    29тЖТ7. Stats bar (overall + per-difficulty solved %)
    30тЖТ    30тЖТ8. Dark/light theme toggle (default dark)
    31тЖТ    31тЖТ9. Empty state, loading states, error toasts
    32тЖТ    32тЖТ10. Replace-existing-sheet flow (preserves solved IDs that still exist in the new CSV)
    33тЖТ    33тЖТ
    34тЖТ    34тЖТ### New functionality the full-stack version adds
    35тЖТ    35тЖТ
    36тЖТ    36тЖТ1. **User accounts** тАФ register, login, logout
    37тЖТ    37тЖТ2. **Server-side persistence** тАФ sheets and solved state live in MongoDB, not localStorage
    38тЖТ    38тЖТ3. **Multi-device sync** тАФ same user logs in from anywhere, sees their data
    39тЖТ    39тЖТ4. **Protected routes** тАФ frontend auth gates; backend 401s unauthenticated requests
    40тЖТ    40тЖТ5. **Token refresh** тАФ short-lived access token (15 min) + long-lived refresh token (7 days), httpOnly cookie for refresh
    41тЖТ    41тЖТ6. **Password security** тАФ bcrypt hashing, never store plaintext
    42тЖТ    42тЖТ7. **Rate limiting** тАФ login/register endpoints limited to prevent brute force
    43тЖТ    43тЖТ8. **CSV size limit** тАФ enforce max ~5 MB per upload server-side
    44тЖТ    44тЖТ

   202тЖТ   202тЖТ### 2.4 Database schema (MongoDB collections)
   203тЖТ   203тЖТ
   204тЖТ   204тЖТUse **three collections**. Use `_id: ObjectId` as the primary key, and store `owner_id` as a string (the user's `_id` hex) on every owned document.
   205тЖТ   205тЖТ
   206тЖТ   206тЖТ#### `users`
   207тЖТ   207тЖТ
   208тЖТ   208тЖТ```json
   209тЖТ   209тЖТ{
   210тЖТ   210тЖТ  "_id": ObjectId,
   211тЖТ   211тЖТ  "email": "alice@example.com",         // unique, lowercased, indexed
   212тЖТ   212тЖТ  "name": "Alice",
   213тЖТ   213тЖТ  "hashed_password": "$2b$12$...",      // bcrypt
   214тЖТ   214тЖТ  "created_at": ISODate,
   215тЖТ   215тЖТ  "updated_at": ISODate
   216тЖТ   216тЖТ}
   217тЖТ   217тЖТ```
   218тЖТ   218тЖТ
   219тЖТ   219тЖТIndexes:
   220тЖТ   220тЖТ- `unique` on `email` (case-insensitive тАФ store lowercased)
   221тЖТ   221тЖТ
   222тЖТ   222тЖТ#### `sheets`
   223тЖТ   223тЖТ
   224тЖТ   224тЖТ```json
   225тЖТ   225тЖТ{
   226тЖТ   226тЖТ  "_id": ObjectId,
   227тЖТ   227тЖТ  "owner_id": "65f...",                 // user._id hex string
   228тЖТ   228тЖТ  "name": "DP",                          // user-facing alias
   229тЖТ   229тЖТ  "created_at": ISODate,
   230тЖТ   230тЖТ  "updated_at": ISODate
   231тЖТ   231тЖТ}
   232тЖТ   232тЖТ```
   233тЖТ   233тЖТ
   234тЖТ   234тЖТIndexes:
   235тЖТ   235тЖТ- `owner_id` (regular)
   236тЖТ   236тЖТ- compound `(owner_id, name)` unique тАФ prevent duplicate sheet names per user
   237тЖТ   237тЖТ
   238тЖТ   238тЖТ#### `questions`
   239тЖТ   239тЖТ
   240тЖТ   240тЖТDenormalized: one document per question, **per sheet**. This avoids the 16 MB document-size limit you'd hit with 700+ embedded questions per sheet, and lets the DB do filtering/sorting/pagination natively.
   241тЖТ   241тЖТ
   242тЖТ   242тЖТ```json
   243тЖТ   243тЖТ{
   244тЖТ   244тЖТ  "_id": ObjectId,
   245тЖТ   245тЖТ  "sheet_id": "65f...",
   246тЖТ   246тЖТ  "owner_id": "65f...",                 // denormalized for auth checks
   247тЖТ   247тЖТ  "leetcode_id": 1,                      // the CSV "ID" column
   248тЖТ   248тЖТ  "url": "https://leetcode.com/problems/two-sum",
   249тЖТ   249тЖТ  "title": "Two Sum",
   250тЖТ   250тЖТ  "difficulty": "Easy",                  // "Easy" | "Medium" | "Hard"
   251тЖТ   251тЖТ  "acceptance": 57.5,                    // float, 0-100
   252тЖТ   252тЖТ  "frequency": 100.0,                    // float, 0-100
   253тЖТ   253тЖТ  "solved": false,                       // bool
   254тЖТ   254тЖТ  "solved_at": null,                     // ISODate or null
   255тЖТ   255тЖТ  "created_at": ISODate
   256тЖТ   256тЖТ}
   257тЖТ   257тЖТ```
   258тЖТ   258тЖТ
   259тЖТ   259тЖТIndexes:
   260тЖТ   260тЖТ- `owner_id`
   261тЖТ   261тЖТ- `sheet_id`
   262тЖТ   262тЖТ- compound `(sheet_id, leetcode_id)` unique тАФ prevents duplicates within a sheet
   263тЖТ   263тЖТ- compound `(sheet_id, difficulty)` тАФ for filter queries
   264тЖТ   264тЖТ- compound `(sheet_id, frequency)` тАФ for range + sort
   265тЖТ   265тЖТ- compound `(sheet_id, acceptance)` тАФ for range + sort
   266тЖТ   266тЖТ- text index on `title` (or use regex substring match тАФ see ┬з2.7)
   267тЖТ   267тЖТ
   268тЖТ   268тЖТ### 2.5 Auth тАФ JWT flow
   269тЖТ   269тЖТ
   270тЖТ   270тЖТ**Two tokens, refresh in httpOnly cookie:**
   271тЖТ   271тЖТ
   272тЖТ   272тЖТ| Token | Lifetime | Storage | Purpose |
   273тЖТ   273тЖТ| --- | --- | --- | --- |
   274тЖТ   274тЖТ| Access token | 15 minutes | Frontend memory (or localStorage) | Sent as `Authorization: Bearer <token>` on every API call |
   275тЖТ   275тЖТ| Refresh token | 7 days | httpOnly cookie named `lt_refresh` | Used by frontend interceptor to silently obtain new access tokens |
   276тЖТ   276тЖТ
   277тЖТ   277тЖТ**Refresh token rotation:** every time `/api/auth/refresh` is called, issue a new refresh token and invalidate the old one (store a `token_id` in the user doc and a `revoked` set тАФ or simpler v1: just rotate without revocation, accept the small risk).
   278тЖТ   278тЖТ
   279тЖТ   279тЖТ#### Endpoints

   313тЖТ   313тЖТ
   314тЖТ   314тЖТ```python
   315тЖТ   315тЖТfrom datetime import datetime, timedelta, timezone
   316тЖТ   316тЖТfrom typing import Any
   317тЖТ   317тЖТfrom jose import jwt, JWTError
   318тЖТ   318тЖТfrom passlib.context import CryptContext
   319тЖТ   319тЖТ
   320тЖТ   320тЖТpwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
   321тЖТ   321тЖТ
   322тЖТ   322тЖТdef hash_password(p: str) -> str:
   323тЖТ   323тЖТ    return pwd_context.hash(p)
   324тЖТ   324тЖТ
   325тЖТ   325тЖТdef verify_password(plain: str, hashed: str) -> bool:
   326тЖТ   326тЖТ    return pwd_context.verify(plain, hashed)
   327тЖТ   327тЖТ
   328тЖТ   328тЖТdef create_access_token(subject: str, expires_minutes: int, secret: str, algo: str) -> str:
   329тЖТ   329тЖТ    expire = datetime.now(timezone.utc) + timedelta(minutes=expires_minutes)
   330тЖТ   330тЖТ    return jwt.encode({"sub": subject, "exp": expire, "type": "access"}, secret, algorithm=algo)
   331тЖТ   331тЖТ
   332тЖТ   332тЖТdef create_refresh_token(subject: str, expires_days: int, secret: str, algo: str) -> str:
   333тЖТ   333тЖТ    expire = datetime.now(timezone.utc) + timedelta(days=expires_days)
   334тЖТ   334тЖТ    return jwt.encode({"sub": subject, "exp": expire, "type": "refresh"}, secret, algorithm=algo)
   335тЖТ   335тЖТ
   336тЖТ   336тЖТdef decode_token(token: str, secret: str, algo: str) -> dict[str, Any] | None:
   337тЖТ   337тЖТ    try:
   338тЖТ   338тЖТ        return jwt.decode(token, secret, algorithms=[algo])
   339тЖТ   339тЖТ    except JWTError:
   340тЖТ   340тЖТ        return None
   341тЖТ   341тЖТ```

   453тЖТ   453тЖТ### 2.7 Reference: building the MongoDB query
   454тЖТ   454тЖТ
   455тЖТ   455тЖТ```python
   456тЖТ   456тЖТ# app/routes/questions.py
   457тЖТ   457тЖТasync def list_questions(sheet_id: str, params: QuestionQueryParams, user, db):
   458тЖТ   458тЖТ    # Ownership check
   459тЖТ   459тЖТ    sheet = await db.sheets.find_one({"_id": ObjectId(sheet_id), "owner_id": str(user["_id"])})
   460тЖТ   460тЖТ    if not sheet:
   461тЖТ   461тЖТ        raise HTTPException(404, "Sheet not found")
   462тЖТ   462тЖТ
   463тЖТ   463тЖТ    query: dict = {"sheet_id": sheet_id, "owner_id": str(user["_id"])}
   464тЖТ   464тЖТ
   465тЖТ   465тЖТ    if params.search:
   466тЖТ   466тЖТ        # Case-insensitive substring тАФ use regex. For larger datasets, switch to
   467тЖТ   467тЖТ        # Atlas Search ($search) with an autocomplete index on `title`.
   468тЖТ   468тЖТ        query["title"] = {"$regex": re.escape(params.search), "$options": "i"}
   469тЖТ   469тЖТ
   470тЖТ   470тЖТ    if params.difficulty:
   471тЖТ   471тЖТ        diffs = [d.strip() for d in params.difficulty.split(",") if d.strip()]
   472тЖТ   472тЖТ        if diffs:
   473тЖТ   473тЖТ            query["difficulty"] = {"$in": diffs}
   474тЖТ   474тЖТ
   475тЖТ   475тЖТ    # Frequency: presets override range
   476тЖТ   476тЖТ    if params.freq_presets:
   477тЖТ   477тЖТ        presets = [int(x) for x in params.freq_presets.split(",") if x.strip().isdigit()]
   478тЖТ   478тЖТ        if presets:
   479тЖТ   479тЖТ            # Match rounded frequency to any preset
   480тЖТ   480тЖТ            query["$or"] = [{"frequency": {"$gte": p - 0.5, "$lt": p + 0.5}} for p in presets]
   481тЖТ   481тЖТ    else:
   482тЖТ   482тЖТ        query["frequency"] = {"$gte": params.freq_min, "$lte": params.freq_max}
   483тЖТ   483тЖТ
   484тЖТ   484тЖТ    if params.acc_min != 0 or params.acc_max != 100:
   485тЖТ   485тЖТ        query["acceptance"] = {"$gte": params.acc_min, "$lte": params.acc_max}
   486тЖТ   486тЖТ
   487тЖТ   487тЖТ    if params.solved == "solved":
   488тЖТ   488тЖТ        query["solved"] = True
   489тЖТ   489тЖТ    elif params.solved == "unsolved":
   490тЖТ   490тЖТ        query["solved"] = False
   491тЖТ   491тЖТ
   492тЖТ   492тЖТ    sort_field = {
   493тЖТ   493тЖТ        "id": "leetcode_id",
   494тЖТ   494тЖТ        "title": "title",
   495тЖТ   495тЖТ        "difficulty": "difficulty",  # store as 0/1/2 in DB or use a sort map
   496тЖТ   496тЖТ        "acceptance": "acceptance",
   497тЖТ   497тЖТ        "frequency": "frequency",
   498тЖТ   498тЖТ    }[params.sort_by]
   499тЖТ   499тЖТ    sort_dir = 1 if params.sort_dir == "asc" else -1
   500тЖТ   500тЖТ
   501тЖТ   501тЖТ    skip = (params.page - 1) * params.page_size
   502тЖТ   502тЖТ
   503тЖТ   503тЖТ    cursor = db.questions.find(query).sort(sort_field, sort_dir).skip(skip).limit(params.page_size)
   504тЖТ   504тЖТ    questions = await cursor.to_list(length=params.page_size)
   505тЖТ   505тЖТ    total = await db.questions.count_documents(query)
   506тЖТ   506тЖТ
   507тЖТ   507тЖТ    # Sheet-level stats (not filtered)
   508тЖТ   508тЖТ    stats = await compute_sheet_stats(sheet_id, db)
   509тЖТ   509тЖТ
   510тЖТ   510тЖТ    return {
   511тЖТ   511тЖТ        "questions": [serialize_question(q) for q in questions],
   512тЖТ   512тЖТ        "total": total,
   513тЖТ   513тЖТ        "page": params.page,
   514тЖТ   514тЖТ        "page_size": params.page_size,
   515тЖТ   515тЖТ        "total_pages": (total + params.page_size - 1) // params.page_size,
   516тЖТ   516тЖТ        "stats": stats,
   517тЖТ   517тЖТ    }
   518тЖТ   518тЖТ```
   519тЖТ   519тЖТ
   520тЖТ   520тЖТ> **Tip:** For difficulty sorting, either (a) store a `difficulty_order` int field (`Easy=0`, `Medium=1`, `Hard=2`) updated on insert, or (b) add a `$addFields` stage in an aggregation pipeline that maps the string to a number. Option (a) is simpler and faster.

   522тЖТ   522тЖТ### 2.8 CSV parsing тАФ server-side
   523тЖТ   523тЖТ
   524тЖТ   524тЖТUse Python's stdlib `csv` module with `io.StringIO`. Stream-parse to handle 5 MB files comfortably.
   525тЖТ   525тЖТ
   526тЖТ   526тЖТ```python
   527тЖТ   527тЖТ# app/services/csv_parser.py
   528тЖТ   528тЖТimport csv, io
   529тЖТ   529тЖТfrom typing import Iterable
   530тЖТ   530тЖТ
   531тЖТ   531тЖТEXPECTED_HEADERS = ["ID", "URL", "Title", "Difficulty", "Acceptance %", "Frequency %"]
   532тЖТ   532тЖТVALID_DIFFICULTIES = {"Easy", "Medium", "Hard"}
   533тЖТ   533тЖТ
   534тЖТ   534тЖТclass CsvValidationError(Exception):
   535тЖТ   535тЖТ    pass
   536тЖТ   536тЖТ
   537тЖТ   537тЖТdef parse_leetcode_csv(content: str) -> tuple[list[dict], int]:
   538тЖТ   538тЖТ    """Returns (questions, skipped_count)."""
   539тЖТ   539тЖТ    reader = csv.DictReader(io.StringIO(content))
   540тЖТ   540тЖТ    headers = [h.strip() for h in (reader.fieldnames or [])]
   541тЖТ   541тЖТ    missing = [h for h in EXPECTED_HEADERS if h not in headers]
   542тЖТ   542тЖТ    if missing:
   543тЖТ   543тЖТ        raise CsvValidationError(f"Missing required columns: {', '.join(missing)}")
   544тЖТ   544тЖТ
   545тЖТ   545тЖТ    questions, skipped = [], 0
   546тЖТ   546тЖТ    for row in reader:
   547тЖТ   547тЖТ        try:
   548тЖТ   548тЖТ            q = {
   549тЖТ   549тЖТ                "leetcode_id": int(row["ID"]),
   550тЖТ   550тЖТ                "url": row["URL"].strip(),
   551тЖТ   551тЖТ                "title": row["Title"].strip(),
   552тЖТ   552тЖТ                "difficulty": row["Difficulty"].strip(),
   553тЖТ   553тЖТ                "acceptance": float(row["Acceptance %"].strip().rstrip("%")),
   554тЖТ   554тЖТ                "frequency": float(row["Frequency %"].strip().rstrip("%")),
   555тЖТ   555тЖТ            }
   556тЖТ   556тЖТ            if q["difficulty"] not in VALID_DIFFICULTIES:
   557тЖТ   557тЖТ                raise ValueError("bad difficulty")
   558тЖТ   558тЖТ            if not q["url"] or not q["title"]:
   559тЖТ   559тЖТ                raise ValueError("empty required field")
   560тЖТ   560тЖТ            questions.append(q)
   561тЖТ   561тЖТ        except (ValueError, KeyError, TypeError):
   562тЖТ   562тЖТ            skipped += 1
   563тЖТ   563тЖТ
   564тЖТ   564тЖТ    return questions, skipped
   565тЖТ   565тЖТ```
   566тЖТ   566тЖТ
   567тЖТ   567тЖТ#### Replace-existing flow (preserves solved state)
   568тЖТ   568тЖТ
   569тЖТ   569тЖТ```python
   570тЖТ   570тЖТ# app/routes/upload.py
   571тЖТ   571тЖТasync def upload_csv(sheet_id: str, file: UploadFile, user, db):
   572тЖТ   572тЖТ    # ... ownership check ...
   573тЖТ   573тЖТ    content = (await file.read()).decode("utf-8", errors="replace")
   574тЖТ   574тЖТ    if len(content) > settings.MAX_CSV_SIZE_BYTES:
   575тЖТ   575тЖТ        raise HTTPException(413, "File too large")
   576тЖТ   576тЖТ
   577тЖТ   577тЖТ    try:
   578тЖТ   578тЖТ        new_questions, skipped = parse_leetcode_csv(content)
   579тЖТ   579тЖТ    except CsvValidationError as e:
   580тЖТ   580тЖТ        raise HTTPException(400, str(e))
   581тЖТ   581тЖТ
   582тЖТ   582тЖТ    if not new_questions:
   583тЖТ   583тЖТ        raise HTTPException(400, "No valid rows in CSV")
   584тЖТ   584тЖТ
   585тЖТ   585тЖТ    # Fetch currently-solved leetcode_ids in this sheet
   586тЖТ   586тЖТ    existing_solved = await db.questions.find(
   587тЖТ   587тЖТ        {"sheet_id": sheet_id, "solved": True},
   588тЖТ   588тЖТ        {"leetcode_id": 1, "_id": 0}
   589тЖТ   589тЖТ    ).to_list(length=None)
   590тЖТ   590тЖТ    solved_set = {q["leetcode_id"] for q in existing_solved}
   591тЖТ   591тЖТ
   592тЖТ   592тЖТ    # Atomic-ish replace: delete all, insert all (use a transaction for safety)
   593тЖТ   593тЖТ    async with await db.client.start_session() as session:
   594тЖТ   594тЖТ        async with session.start_transaction():
   595тЖТ   595тЖТ            await db.questions.delete_many({"sheet_id": sheet_id}, session=session)
   596тЖТ   596тЖТ            docs = [
   597тЖТ   597тЖТ                {
   598тЖТ   598тЖТ                    **q,
   599тЖТ   599тЖТ                    "sheet_id": sheet_id,
   600тЖТ   600тЖТ                    "owner_id": str(user["_id"]),
   601тЖТ   601тЖТ                    "solved": q["leetcode_id"] in solved_set,
   602тЖТ   602тЖТ                    "solved_at": None,
   603тЖТ   603тЖТ                    "difficulty_order": {"Easy": 0, "Medium": 1, "Hard": 2}[q["difficulty"]],
   604тЖТ   604тЖТ                    "created_at": datetime.now(timezone.utc),
   605тЖТ   605тЖТ                }
   606тЖТ   606тЖТ                for q in new_questions
   607тЖТ   607тЖТ            ]
   608тЖТ   608тЖТ            if docs:
   609тЖТ   609тЖТ                await db.questions.insert_many(docs, session=session)
   610тЖТ   610тЖТ            await db.sheets.update_one(
   611тЖТ   611тЖТ                {"_id": ObjectId(sheet_id)},
   612тЖТ   612тЖТ                {"$set": {"updated_at": datetime.now(timezone.utc)}},
   613тЖТ   613тЖТ                session=session,
   614тЖТ   614тЖТ            )
   615тЖТ   615тЖТ
   616тЖТ   616тЖТ    preserved = sum(1 for q in new_questions if q["leetcode_id"] in solved_set)
   617тЖТ   617тЖТ    return {"imported": len(new_questions), "skipped": skipped, "preserved_solved": preserved}
   618тЖТ   618тЖТ```

   820тЖТ   820тЖТ### 3.5 API client тАФ axios with refresh-on-401 interceptor
   821тЖТ   821тЖТ
   822тЖТ   822тЖТThis is the most important piece of frontend auth logic. Single-flight refresh (don't fire 10 refresh calls when 10 parallel requests all 401).
   823тЖТ   823тЖТ
   824тЖТ   824тЖТ```ts
   825тЖТ   825тЖТ// src/api/client.ts
   826тЖТ   826тЖТimport axios, { AxiosError, InternalAxiosRequestConfig } from "axios";
   827тЖТ   827тЖТ
   828тЖТ   828тЖТconst API_URL = import.meta.env.VITE_API_URL;
   829тЖТ   829тЖТ
   830тЖТ   830тЖТlet accessToken: string | null = null;
   831тЖТ   831тЖТlet refreshPromise: Promise<string> | null = null;
   832тЖТ   832тЖТ
   833тЖТ   833тЖТexport function setAccessToken(t: string | null) {
   834тЖТ   834тЖТ  accessToken = t;
   835тЖТ   835тЖТ}
   836тЖТ   836тЖТ
   837тЖТ   837тЖТexport const api = axios.create({ baseURL: `${API_URL}/api`, withCredentials: true });
   838тЖТ   838тЖТ
   839тЖТ   839тЖТapi.interceptors.request.use((config: InternalAxiosRequestConfig) => {
   840тЖТ   840тЖТ  if (accessToken) {
   841тЖТ   841тЖТ    config.headers.Authorization = `Bearer ${accessToken}`;
   842тЖТ   842тЖТ  }
   843тЖТ   843тЖТ  return config;
   844тЖТ   844тЖТ});
   845тЖТ   845тЖТ
   846тЖТ   846тЖТapi.interceptors.response.use(
   847тЖТ   847тЖТ  (r) => r,
   848тЖТ   848тЖТ  async (error: AxiosError) => {
   849тЖТ   849тЖТ    const original = error.config as InternalAxiosRequestConfig & { _retry?: boolean };
   850тЖТ   850тЖТ    if (error.response?.status === 401 && !original._retry && !original.url?.includes("/auth/")) {
   851тЖТ   851тЖТ      original._retry = true;
   852тЖТ   852тЖТ      try {
   853тЖТ   853тЖТ        if (!refreshPromise) {
   854тЖТ   854тЖТ          refreshPromise = api.post("/auth/refresh").then((r) => r.data.access_token);
   855тЖТ   855тЖТ        }
   856тЖТ   856тЖТ        const newToken = await refreshPromise;
   857тЖТ   857тЖТ        refreshPromise = null;
   858тЖТ   858тЖТ        setAccessToken(newToken);
   859тЖТ   859тЖТ        original.headers.Authorization = `Bearer ${newToken}`;
   860тЖТ   860тЖТ        return api(original);
   861тЖТ   861тЖТ      } catch {
   862тЖТ   862тЖТ        refreshPromise = null;
   863тЖТ   863тЖТ        setAccessToken(null);
   864тЖТ   864тЖТ        window.location.href = "/login";
   865тЖТ   865тЖТ        return Promise.reject(error);
   866тЖТ   866тЖТ      }
   867тЖТ   867тЖТ    }
   868тЖТ   868тЖТ    return Promise.reject(error);
   869тЖТ   869тЖТ  },
   870тЖТ   870тЖТ);
   871тЖТ   871тЖТ```
   872тЖТ   872тЖТ
   873тЖТ   873тЖТ### 3.6 Auth context
   874тЖТ   874тЖТ
   875тЖТ   875тЖТ```tsx
   876тЖТ   876тЖТ// src/context/AuthContext.tsx
   877тЖТ   877тЖТimport { createContext, useContext, useEffect, useState, ReactNode } from "react";
   878тЖТ   878тЖТimport { api, setAccessToken } from "@/api/client";
   879тЖТ   879тЖТ
   880тЖТ   880тЖТinterface User { id: string; email: string; name: string; }
   881тЖТ   881тЖТinterface AuthCtx {
   882тЖТ   882тЖТ  user: User | null;
   883тЖТ   883тЖТ  loading: boolean;
   884тЖТ   884тЖТ  login: (email: string, password: string) => Promise<void>;
   885тЖТ   885тЖТ  register: (email: string, password: string, name: string) => Promise<void>;
   886тЖТ   886тЖТ  logout: () => Promise<void>;
   887тЖТ   887тЖТ}
   888тЖТ   888тЖТ
   889тЖТ   889тЖТconst Ctx = createContext<AuthCtx>(null!);
   890тЖТ   890тЖТexport const useAuth = () => useContext(Ctx);
   891тЖТ   891тЖТ
   892тЖТ   892тЖТexport function AuthProvider({ children }: { children: ReactNode }) {
   893тЖТ   893тЖТ  const [user, setUser] = useState<User | null>(null);
   894тЖТ   894тЖТ  const [loading, setLoading] = useState(true);
   895тЖТ   895тЖТ
   896тЖТ   896тЖТ  // On mount: try to get a fresh access token via refresh cookie
   897тЖТ   897тЖТ  useEffect(() => {
   898тЖТ   898тЖТ    (async () => {
   899тЖТ   899тЖТ      try {
   900тЖТ   900тЖТ        const r = await api.post("/auth/refresh");
   901тЖТ   901тЖТ        setAccessToken(r.data.access_token);
   902тЖТ   902тЖТ        const me = await api.get("/auth/me");
   903тЖТ   903тЖТ        setUser(me.data);
   904тЖТ   904тЖТ      } catch {
   905тЖТ   905тЖТ        setAccessToken(null);
   906тЖТ   906тЖТ        setUser(null);
   907тЖТ   907тЖТ      } finally {
   908тЖТ   908тЖТ        setLoading(false);
   909тЖТ   909тЖТ      }
   910тЖТ   910тЖТ    })();
   911тЖТ   911тЖТ  }, []);
   912тЖТ   912тЖТ
   913тЖТ   913тЖТ  const login = async (email: string, password: string) => {
   914тЖТ   914тЖТ    const r = await api.post("/auth/login", { email, password });
   915тЖТ   915тЖТ    setAccessToken(r.data.access_token);
   916тЖТ   916тЖТ    setUser(r.data.user);
   917тЖТ   917тЖТ  };
   918тЖТ   918тЖТ
   919тЖТ   919тЖТ  const register = async (email: string, password: string, name: string) => {
   920тЖТ   920тЖТ    const r = await api.post("/auth/register", { email, password, name });
   921тЖТ   921тЖТ    setAccessToken(r.data.access_token);
   922тЖТ   922тЖТ    setUser(r.data.user);
   923тЖТ   923тЖТ  };
   924тЖТ   924тЖТ
   925тЖТ   925тЖТ  const logout = async () => {
   926тЖТ   926тЖТ    await api.post("/auth/logout");
   927тЖТ   927тЖТ    setAccessToken(null);
   928тЖТ   928тЖТ    setUser(null);
   929тЖТ   929тЖТ  };
   930тЖТ   930тЖТ
   931тЖТ   931тЖТ  return <Ctx.Provider value={{ user, loading, login, register, logout }}>{children}</Ctx.Provider>;
   932тЖТ   932тЖТ}
   933тЖТ   933тЖТ```

   989тЖТ   989тЖТ```ts
   990тЖТ   990тЖТ// src/hooks/useSheets.ts

   996тЖТ   996тЖТ  return useQuery({
   997тЖТ   997тЖТ    queryKey: ["sheets"],

   999тЖТ   999тЖТ  });

  1002тЖТ  1002тЖТexport function useCreateSheet() {

  1010тЖТ  1010тЖТexport function useDeleteSheet() {
  1011тЖТ  1011тЖТ  const qc = useQueryClient();
  1012тЖТ  1012тЖТ  return useMutation({
  1013тЖТ  1013тЖТ    mutationFn: async (id: string) => api.delete(`/sheets/${id}`),
  1014тЖТ  1014тЖТ    onSuccess: () => qc.invalidateQueries({ queryKey: ["sheets"] }),
  1015тЖТ  1015тЖТ  });

  1017тЖТ  1017тЖТ
  1018тЖТ  1018тЖТexport function useQuestions(sheetId: string | null, query: QuestionQuery) {
  1019тЖТ  1019тЖТ  return useQuery({
  1020тЖТ  1020тЖТ    queryKey: ["questions", sheetId, query],
  1021тЖТ  1021тЖТ    queryFn: async () => {
  1022тЖТ  1022тЖТ      if (!sheetId) return null;
  1023тЖТ  1023тЖТ      const params = new URLSearchParams();
  1024тЖТ  1024тЖТ      Object.entries(query).forEach(([k, v]) => params.append(k, String(v)));
  1025тЖТ  1025тЖТ      return (await api.get(`/sheets/${sheetId}/questions?${params}`)).data;
  1026тЖТ  1026тЖТ    },
  1027тЖТ  1027тЖТ    enabled: !!sheetId,
  1028тЖТ  1028тЖТ  });
  1029тЖТ  1029тЖТ}
  1030тЖТ  1030тЖТ
  1031тЖТ  1031тЖТexport function useToggleSolved(sheetId: string) {
  1032тЖТ  1032тЖТ  const qc = useQueryClient();
  1033тЖТ  1033тЖТ  return useMutation({
  1034тЖТ  1034тЖТ    mutationFn: async ({ questionId, solved }: { questionId: string; solved: boolean }) =>
  1035тЖТ  1035тЖТ      (await api.patch(`/sheets/${sheetId}/questions/${questionId}`, { solved })).data.question,
  1036тЖТ  1036тЖТ    // Optimistic update: see TanStack Query docs
  1037тЖТ  1037тЖТ  });
  1038тЖТ  1038тЖТ}
  1039тЖТ  1039тЖТ
  1040тЖТ  1040тЖТexport function useUploadCsv() {
  1041тЖТ  1041тЖТ  const qc = useQueryClient();
  1042тЖТ  1042тЖТ  return useMutation({
  1043тЖТ  1043тЖТ    mutationFn: async ({ sheetId, file, mode, alias }: {
  1044тЖТ  1044тЖТ      sheetId?: string; file: File; mode: "create" | "replace"; alias: string;
  1045тЖТ  1045тЖТ    }) => {
  1046тЖТ  1046тЖТ      // If create mode: first create the sheet, then upload
  1047тЖТ  1047тЖТ      let id = sheetId;
  1048тЖТ  1048тЖТ      if (mode === "create") {
  1049тЖТ  1049тЖТ        const created = (await api.post("/sheets", { name: alias })).data.sheet;
  1050тЖТ  1050тЖТ        id = created.id;
  1051тЖТ  1051тЖТ      }
  1052тЖТ  1052тЖТ      const fd = new FormData();
  1053тЖТ  1053тЖТ      fd.append("file", file);
  1054тЖТ  1054тЖТ      const r = await api.post(`/sheets/${id}/upload`, fd);
  1055тЖТ  1055тЖТ      qc.invalidateQueries({ queryKey: ["sheets"] });
  1056тЖТ  1056тЖТ      qc.invalidateQueries({ queryKey: ["questions"] });
  1057тЖТ  1057тЖТ      return r.data;
  1058тЖТ  1058тЖТ    },
  1059тЖТ  1059тЖТ  });
  1060тЖТ  1060тЖТ}

  1168тЖТ  1168тЖТ
  1169тЖТ  1169тЖТ```python
  1170тЖТ  1170тЖТresponse.set_cookie(
  1171тЖТ  1171тЖТ    key="lt_refresh",
